## [1매] 개요 — AS-IS 진단 · 개선 전략 · KPI

### 현행 플로우와 이탈 구간(가설)
```
시작 ─▶ 본인인증 ─▶ 약관동의/정보입력 ─▶ (분석) ─▶ 진단결과 ─▶ 추천상품 ─▶ 상담신청
 │          │              │                          │           │          ▲
 이탈①      이탈②          이탈③                      이탈④       이탈⑤   ◀ 최종 전환(목표)
"이거 왜   "인증 귀찮"   "입력 많고          "결과가 어려움/   "내 얘기    상담신청 CTA
 해야하나"              불안(개인정보)"       추상적"          아님"        도달·클릭
```

### 핵심 문제(근거 기반, 신규 아이디어 아님 — H1 충족)
| # | AS-IS 문제(가설) | 근거 |
| --- | --- | --- |
| D1 | **시작 동기 부족** — 무엇을 얻는지·얼마나 걸리는지 불명확 | 금융 진단형 퍼널 공통 이탈 패턴, 진행률 부재 |
| D2 | **본인인증 마찰** — 초반 강제 인증으로 진입 전 이탈 | 보험/금융 모바일 가입 1차 이탈 최대 구간 |
| D3 | **입력 부담·개인정보 불안** — 한 화면 다량 입력 | 폼 길이↔완료율 반비례(업계 통설) |
| D4 | **결과의 추상성** — 보장 갭이 한눈에 안 들어옴 | 숫자 나열형 결과의 낮은 행동 전환 |
| D5 | **상담신청까지 거리** — 결과→상담 동선 단절 | 결과 화면 내 약한 CTA |

### 개선 전략(TO-BE 원칙)
1. **선(先)가치-후(後)인증** — 인증을 결과 직전/상담 단계로 지연(D2).
2. **여정 가시화** — 진행률·예상 소요시간 상시 노출(D1·D3).
3. **갭의 시각화** — "내 보장 부족분"을 그래프/금액으로 즉시 체감(D4).
4. **결과=행동 연결** — 결과 카드마다 상담 CTA를 밀착(D5).
5. **신뢰 장치** — 개인정보 사용 목적·암호화·"가입 권유 아님" 고지로 불안 해소(D3).

### 성공지표(KPI)
| 지표 | 정의 | 목표(가설) |
| --- | --- | --- |
| 진단 완료율 | 시작 → 진단결과 도달 | +15%p |
| 상담신청 전환율 | 결과 도달 → 상담신청 | +8%p |
| 단계별 이탈율 | 각 스텝 이탈 | 최대 이탈 구간 −20% |
| 본인인증 통과율 | 인증 시작 → 완료 | +10%p |

> 측정: Performance Lab(R-008) 연동, 단계별 퍼널 + A/B 테스트로 검증.

## [2매] SB-01 · 보장분석 시작(랜딩)

```
┌───────────────────────────┐
│  ← 보장분석            ⓘ   │  상단: 뒤로/도움말
│                           │
│  3분이면 끝나는            │  ① 가치 헤드라인(소요시간 명시)
│  내 보장 빈틈 진단         │
│                           │
│  ▥ 또래 평균 대비          │  ② 미리보기(개인정보 입력 전
│     내 보장은?            │     기대 결과 예고 — 동기부여)
│  [■■■□□ 65%]              │
│                           │
│  ✓ 가입 권유 아님          │  ③ 신뢰 배지(불안 해소)
│  ✓ 정보는 진단에만 사용     │
│  ✓ 1분 입력 · 인증 없이 시작│
│                           │
│  ┌─────────────────────┐  │
│  │   무료로 진단 시작 ▶  │  │  ④ 주 CTA(풀폭·고대비)
│  └─────────────────────┘  │
│  로그인 회원은 자동입력 →  │  ⑤ 보조 동선
└───────────────────────────┘
```
| 구성요소 | 디스크립션(기능정의) |
| --- | --- |
| ① 가치 헤드라인 | 소요시간·결과물 명시. "3분/내 보장 빈틈" 등 손실회피 프레이밍 |
| ② 결과 미리보기 | 입력 전 또래 평균 대비 게이지 샘플 노출로 호기심·기대 형성(D1) |
| ③ 신뢰 배지 | "가입 권유 아님/정보 진단용/인증 없이 시작" 3종(D2·D3) |
| ④ 주 CTA | 풀폭 버튼, 명도대비 4.5:1↑, 탭영역 ≥44px(WCAG 2.2 AA) |
| ⑤ 회원 동선 | 로그인 시 보유정보 자동입력으로 입력 단축 |

**AS-IS→TO-BE:** 막연한 시작 → 가치·소요·신뢰 제시 + **인증 없이 시작**(D1·D2). **전환 기여:** 진입 이탈↓.

## [4매] SB-03 · 분석 진행 & 본인인증(지연 배치)

```
┌───────────────────────────┐
│        분석 중…            │  ① 진행 애니메이션
│   또래 12,840명과 비교 중   │     (대기=가치 가공 인식)
│   [■■■■■■■■□] 80%          │
│                           │
│  결과를 안전하게 보려면     │  ② 인증을 "결과 직전"에
│  본인 확인이 필요해요       │     배치(선가치-후인증)
│                           │
│  [ 간편인증(카카오/네이버/  │  ③ 간편인증 우선 노출
│    토스/PASS) ]            │     (현행 결제수단과 일관 R-009)

│                           │
│  왜 필요한가요? ⌄          │  ④ 인라인 설명(신뢰)

│  · 가입 권유 전화 없음      │
└───────────────────────────┘
```
| 구성요소 | 디스크립션 |
| --- | --- |
| ① 진행 표현 | 분석 대기 시간을 "또래 비교 중" 메시지로 가치화(체감 단축) |
| ② 인증 지연 | 시작이 아닌 **결과 직전**에 인증 → 진입 이탈 최소화(D2) |

**AS-IS→TO-BE:** 초반 강제 인증 → **간편인증을 결과 직전에**(D2). **전환 기여:** 인증 통과율↑·이탈②↓.

## 부록 A. 접근성·반응형 체크(WCAG 2.2 AA)
- 명도대비 본문 4.5:1·대형 3:1↑, 터치 타깃 ≥44×44px, 포커스 가시화.
- 진행률·오류는 색+텍스트 병기(색 의존 금지), 스크린리더 단계 안내(aria-current).
- 단일 컬럼·하단 고정 CTA·엄지영역 배치(모바일 우선).

## 부록 B. 측정·검증 계획(타사 비교 포함 — RFP 허용)
- **퍼널 계측:** 각 SB 단계 진입/완료 이벤트 → Performance Lab 대시보드.
- **A/B 테스트:** ①인증 지연(SB-03) ②갭 시각화(SB-04) 2개 가설 우선 검증.
- **벤치마크:** 동종 다이렉트 보험/핀테크 진단 퍼널의 "선가치-후인증"·"갭 시각화" 사례 비교(설계 근거 명시).

> 본 SB는 신규 아이디어 공모가 아니라 **현행 보장분석 플로우(R-005)의 전환 병목을 근거로 재설계**한 것으로, 제안사의 보험·마케팅 이해와 협업 적합성(H1)을 보여주는 데 목적이 있다. 6매 분량 내 PT용 요약 1매(개요)로 구성.
