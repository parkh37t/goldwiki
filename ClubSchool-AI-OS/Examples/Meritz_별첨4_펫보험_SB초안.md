## 1. AS-IS 진단 — 단계별 이탈 지점(가설)

```
상품소개 ─▶ 정보입력 ─▶ 견적확인 ─▶ 플랜선택 ─▶ 가입정보/약관 ─▶ 결제 ─▶ 완료
  이탈①      이탈②       이탈③       이탈④       이탈⑤          이탈⑥
"가격 모름" "입력 많음"  "견적 늦음"  "비교 어려움" "약관 복잡"     "결제 이탈"
```

| # | 이탈 원인(가설) | 근거 |
| --- | --- | --- |
| D1 | **가격 불투명** — 끝까지 가야 보험료를 앎 | 다이렉트 보험 가입 이탈 1위 요인(가격 사전노출 부재) |
| D2 | **입력 피로** — 반려동물 정보 다량 입력 | 폼 길이↔완료율 반비례 |
| D3 | **견적 지연·플랜 비교 부재** | 단일 플랜 제시 시 의사결정 지연 |
| D4 | **진행 손실** — 이탈 시 처음부터 | 저장 부재로 재방문 가입 단절 |
| D5 | **약관·결제 마찰** | 긴 약관, 결제 단계 이탈 |

## 2. TO-BE 4원칙
1. **즉시 견적·투명 가격** — 3개 입력만으로 바로 보험료 노출(D1·D3).
2. **1스텝 1과제 + 칩 입력** — 타이핑 최소화(D2).
3. **진행 자동저장** — 이탈해도 이어서 가입(D4).
4. **플랜 비교 + 실시간 가격** — 특약 토글 시 즉시 반영(D3·D5).

## 3. 화면 구성(6화면 — 프로토타입과 1:1)

| # | 화면 | 핵심 개선 | 이탈 대응 |
| --- | --- | --- | --- |
| 1 | 상품소개/시작 | "월 1만원대" 가치 + 즉시견적/자동저장/간편가입 배지 | D1 |
| 2 | 반려동물 정보 | 종·나이·품종 **칩 선택**(건너뛰기 허용) + 자동저장 배너 | D2·D4 |
| 3 | 즉시 견적 + 플랜 | 입력 직후 **보험료 노출** + 3플랜 비교(추천 배지) | D1·D3 |
| 4 | 보장 커스터마이즈 | 특약 토글 + **하단 실시간 가격바** | D3 |
| 5 | 가입정보·약관 | 인증 자동채움 + 필수/선택 약관 분리 | D2·D5 |
| 6 | 결제·완료 | 현행 간편결제(토스/네이버/카카오/페이코) 일관 UX | D5 |

## 4. KPI / 검증
| 지표 | 목표(가설) |
| --- | --- |
| 가입 완료율 | +12%p |
| 견적 → 가입 전환 | +10%p |
| 단계별 이탈율 | −25% |

검증: 퍼널 계측(Performance Lab) + A/B(즉시견적·진행저장 2가설) + 타사 펫보험 가입 퍼널 벤치마크. 접근성 WCAG 2.2 AA(탭타깃 48px·명도대비).

> 본 과제는 현행 펫보험 가입 플로우(R-003)의 이탈 병목을 근거로 재설계한 것으로, 보험·마케팅 이해와 협업 적합성(H1)을 증빙한다. SB 6매 이내 구성.
