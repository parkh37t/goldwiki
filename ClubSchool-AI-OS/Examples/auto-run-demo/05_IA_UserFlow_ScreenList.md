## 0. 설계 원칙 (RFP 동인 → IA 결정)

본 IA·플로우는 RFP 분석에서 확정된 발주처 동인 3축과 4개 사용자 역할을 구조로 옮긴 결과다. 모든 결정은 추적 가능한 요구(REQ)에 근거한다.

| 설계 원칙 | 근거(REQ/RFP 신호) | IA·플로우 반영 |
| --- | --- | --- |
| 모바일 퍼스트·도달 깊이 ≤ 3 | REQ-014, H-1(청소년 70% 모바일) | 하단 탭 5개 중심 IA, 핵심 과업 3탭 이내 도달 |
| 역할별 분리 IA(4역할) | REQ-021, RFP §2 | 청소년/지도교사·멘토/기관담당/진흥원관리자 4개 내비 트리 분기 |
| 접근성 내장(KWCAG 2.2 AA) | REQ-013, H-8 | 모든 화면 상태에 포커스·대체텍스트·오류 안내 규격, 저대역폭 폴백 |
| 증빙 신뢰성 가시화 | REQ-004, H-3 | 증빙 발급/검증을 독립 화면(SCR-022/SCR-023)으로 분리 |
| 아동 개인정보 보호 가시화 | REQ-008, RISK-01 | 법정대리인 동의를 별도 화면(SCR-003)·별도 플로우로 격리 |
| 마감일 피크 흡수 | REQ-019, RISK-02 | 지원금 신청 플로우에 대기열·임시저장·재진입 노드 명시 |

> 라벨링 원칙(IA 가이드 §2): 사용자 용어 우선("우리 동아리", "활동일지", "증빙 발급"), 내부 용어("워크플로우", "엔티티") 노출 금지. 카드소팅은 폐쇄형으로 청소년 20명·지도교사 10명 표본 검증을 시행하며, 트리 테스트 탐색 성공률 목표 ≥ 80%(REQ-021 검증, TC-121 시드).

## 2. 핵심 유저플로우 (mermaid)

> 표기 원칙(UserFlowGuide §표기법): 한 화면 = 한 노드, 모든 분기 2출구 이상, 모든 에러에 (메시지·복구·시스템처리). 점선은 백그라운드 시스템 처리. 각 플로우는 화면목록(§3)·QA 시드(§4)와 1:1 정합.

### FLOW-04 회원가입·본인확인 (REQ-007~009 · TC-107~109)

만 14세 미만 분기를 별도 동의 경로로 격리(RISK-01 완화). 데드엔드 없음.

```mermaid
flowchart TD
  S((시작)) --> L["SCR-001 로그인/소셜"]
  L --> SOC{소셜 인증 성공?}
  SOC -- 아니오 --> ERR1["오류: 인증 실패 안내<br/>복구: 다른 수단 재시도"]
  ERR1 --> L
  SOC -- 예 --> IDV["SCR-002 본인확인(실명·연령)"]
  IDV -.본인확인기관 연계.-> IDVSYS[("PASS/실명확인 API")]
  IDVSYS --> AGE{만 14세 미만?}
  AGE -- 아니오 --> AGREE["약관·개인정보 동의"]
  AGE -- 예 --> GUARD["SCR-003 법정대리인 동의 요청"]
  GUARD --> GV{보호자 동의 완료?}
  GV -- 아니오 --> GWAIT["대기 상태 저장<br/>복구: 보호자 재발송/알림"]
  GWAIT -.동의링크 재발송.-> GUARD
  GV -- 예 --> AGREE
  AGREE --> AGV{필수 동의 체크?}
  AGV -- 아니오 --> AGREE
  AGV -- 예 --> DONE["SCR-004 가입 완료"]
  DONE --> ROLE["SCR-005 역할 선택·온보딩"]
  ROLE --> H["SCR-010 홈"]
  IDVSYS -. 타임아웃/5xx .-> TO["오류: 잠시 후 재시도<br/>임시저장 유지"]
  TO --> IDV
  H --> O((종료))
```

### FLOW-01 동아리 검색 → 상세 → 가입(승인) (REQ-001~002 · TC-101~102)

```mermaid
flowchart TD
  S((시작)) --> SR["SCR-011 검색·필터<br/>(지역/관심사/연령/요일)"]
  SR --> RES{검색 결과 있음?}
  RES -- 아니오 --> EMP["SCR-090 결과 없음<br/>복구: 필터 완화 추천"]
  EMP --> SR
  RES -- 예 --> DET["SCR-012 동아리 상세"]
  DET --> CAP{모집 중 & 정원 여유?}
  CAP -- 아니오 --> WAIT["관심 등록/대기 신청 안내"]
  WAIT --> DET
  CAP -- 예 --> APPLY["SCR-013 가입 신청서 제출"]
  APPLY -.신청 접수.-> NOTI[("지도교사 알림 발송")]
  NOTI --> REV["SCR-060 지도교사 가입승인함"]
  REV --> DEC{승인/반려?}
  DEC -- 승인 --> JOINED["가입 완료·우리동아리 편입"]
  DEC -- 반려 --> REJ["반려 사유 안내<br/>복구: 재신청 가능"]
  REJ --> SR
  JOINED --> MY["SCR-020 활동 홈"]
  MY --> O((종료))
```

### FLOW-02 활동기록 → 증빙 발급 (REQ-003~004 · TC-103~104)

증빙 위변조 방지(발급번호·QR·전자서명)를 시스템 처리로 명시(H-3, RISK-06).

```mermaid
flowchart TD
  S((시작)) --> MY["SCR-020 활동 홈"]
  MY --> NEW["SCR-021 활동일지 작성<br/>(일지/사진/봉사시간)"]
  NEW --> VAL{필수값·형식 유효?}
  VAL -- 아니오 --> NEW
  VAL -- 예 --> SAVE["임시저장/제출"]
  SAVE -.업로드.-> UP[("사진·첨부 저장")]
  UP -. 용량초과/형식오류 .-> UPERR["오류: 허용 형식 안내<br/>복구: 재첨부"]
  UPERR --> NEW
  UP --> APPR["SCR-061 지도교사 활동 검토"]
  APPR --> AOK{확인/보완요청?}
  AOK -- 보완요청 --> FIX["보완 사유 안내<br/>복구: 일지 수정 재제출"]
  FIX --> NEW
  AOK -- 확인 --> REQDOC["SCR-022 증빙 발급 요청"]
  REQDOC -.증빙 생성.-> GEN[("발급번호+QR+전자서명 PDF")]
  GEN --> PDF["증빙 PDF 다운로드/공유"]
  PDF --> VERIFY["SCR-023 증빙 검증(공개 페이지)"]
  GEN -. 생성 실패 .-> GENERR["오류: 발급 지연 안내<br/>복구: 재요청/문의"]
  GENERR --> REQDOC
  VERIFY --> O((종료))
```

### FLOW-03 지원금 신청 → 심사 → 정산 (REQ-005~006, 022 · TC-105~106, 122)

마감일 동시접속 피크에 대한 대기열·임시저장·재진입을 명시(REQ-019, RISK-02). 전자서명 부인방지(REQ-006).

```mermaid
flowchart TD
  S((시작)) --> LIST["SCR-030 지원사업 목록"]
  LIST --> ELIG{신청 자격·기간 충족?}
  ELIG -- 아니오 --> NOTELIG["자격 미달/기간 안내<br/>복구: 다른 사업 안내"]
  NOTELIG --> LIST
  ELIG -- 예 --> QUEUE{마감일 피크 대기열?}
  QUEUE -- 대기 --> WAIT["대기 순번 안내<br/>복구: 임시저장 후 재진입"]
  WAIT --> QUEUE
  QUEUE -- 진입 --> FORM["SCR-031 신청서 작성<br/>(자동 임시저장)"]
  FORM --> FVAL{필수값·서식 유효?}
  FVAL -- 아니오 --> FORM
  FVAL -- 예 --> ATT["SCR-032 첨부서류·전자서명"]
  ATT --> SIGN{전자서명 성공?}
  SIGN -- 아니오 --> SIGNERR["오류: 서명 실패 안내<br/>복구: 재서명/수단변경"]
  SIGNERR --> ATT
  SIGN -- 예 --> SUBMIT["신청 제출"]
  SUBMIT -.접수·부인방지 로그.-> LOG[("제출 타임스탬프·서명검증")]
  LOG --> SCR["SCR-070 기관 심사"]
  SCR --> JUDGE{선정/반려/보완?}
  JUDGE -- 보완 --> AMEND["보완 요청 안내<br/>복구: 신청서 수정 재제출"]
  AMEND --> FORM
  JUDGE -- 반려 --> REJ["반려 사유·이의신청 안내"]
  REJ --> RESULT
  JUDGE -- 선정 --> GRANT["교부 결정·SCR-033 결과 확인"]
  GRANT --> SETTLE["SCR-034 정산 보고서 작성"]
  SETTLE --> SVAL{집행 증빙·금액 정합?}
  SVAL -- 아니오 --> SETTLE
  SVAL -- 예 --> SREV["SCR-071 기관 정산 검토"]
  SREV -.회계 추적 로그.-> ACC[("집행 추적·감사 로그")]
  ACC --> RESULT["SCR-033 신청·정산 현황"]
  RESULT --> O((종료))
```
