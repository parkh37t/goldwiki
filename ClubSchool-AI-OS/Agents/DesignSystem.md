# DesignSystem — 디자인시스템팀 역할 카탈로그

> 디자인 토큰·컴포넌트·문서·접근성으로 일관성과 확장성을 보장하는 사람용 역할 카탈로그입니다.
> 실행 정본은 `../.claude/agents/design-system-lead.md`, 거버넌스·표준 정본은
> [GoldWiki/DesignSystem](../GoldWiki/DesignSystem/README.md)입니다.

## 팀 개요

디자인시스템팀은 디자인과 코드를 잇는 단일 진실 공급원을 운영합니다. 토큰·컴포넌트·패턴·
문서를 표준화해 UI·프론트팀이 빠르고 일관되게 제품을 만들도록 하며, 접근성(WCAG)과
버전 관리(SemVer)를 시스템에 내장합니다. 모든 변경은
[DesignSystem](../GoldWiki/DesignSystem/README.md) 거버넌스를 따르고 영향 범위를 기록합니다.

- 핵심 산출물: 디자인 토큰, 컴포넌트 라이브러리, 패턴/가이드라인, 시스템 문서, 접근성 기준
- 주요 협업 팀: UI 디자인팀, 브랜드경험팀, 퍼블리싱·프론트팀, QA팀
- 핵심 문서: [DesignSystem](../GoldWiki/DesignSystem/README.md), [UI](../GoldWiki/UI/README.md), [Frontend](../GoldWiki/Frontend/README.md)

---

## 디자인 시스템 아키텍트 (Design System Architect)

- 미션: 디자인 시스템의 구조·원칙·로드맵·거버넌스를 정의해 확장 가능하고 신뢰할 수 있는 시스템을 보장한다.
- 주요 책임: 시스템 아키텍처/계층(토큰·컴포넌트·패턴) 설계 / 거버넌스/기여 모델 / 버전 전략(SemVer) / 채택률 관리 / 변경 영향 통제.
- 입력: 브랜드 가이드, UI 패턴, 프론트 구현 제약, 사용 현황.
- 출력: 시스템 아키텍처, 거버넌스 정책, 로드맵, 기여 가이드, 버전 정책.
- 협업 대상: UI 디자인 리드, 토큰 엔지니어, 프론트 리드, BX 디렉터.
- 품질 기준: 계층 일관, 버전 규율(SemVer) 준수, 변경 영향 추적, 채택률 측정.

## 토큰 엔지니어 (Design Token Engineer)

- 미션: 컬러·타이포·간격·모션 등 디자인 결정을 플랫폼 중립 토큰으로 체계화해 디자인-코드 동기화를 보장한다.
- 주요 책임: 토큰 구조/네이밍 정의 / 테마/모드(라이트·다크) 토큰 / 토큰 파이프라인(Style Dictionary 등) / 플랫폼별 산출 / 토큰 변경 관리.
- 입력: 브랜드 컬러/타이포, UI 시안, 플랫폼 요건, 모션 스펙.
- 출력: 토큰 정의(JSON 등), 테마 세트, 빌드 산출물, 토큰 문서.
- 협업 대상: DS 아키텍트, 비주얼/모션 디자이너, 프론트 리드, 접근성 스페셜리스트.
- 품질 기준: 네이밍 일관, 단일 출처, 대비(WCAG) 충족, 플랫폼 산출 정확.

## 컴포넌트 디자이너 (Component Designer)

- 미션: 재사용 가능한 컴포넌트를 상태·변형까지 완결되게 설계해 제품 전반의 일관성과 속도를 높인다.
- 주요 책임: 컴포넌트 명세(anatomy·variants·states) / 사용 규칙/도/돈트 / 컴포넌트 토큰 매핑 / API/속성 정의 / 접근성 동작 명세.
- 입력: UI 패턴, 토큰, 사용 사례, 접근성 기준.
- 출력: 컴포넌트 라이브러리, 컴포넌트 명세서, 사용 가이드, 변형/상태 정의.
- 협업 대상: 비주얼/인터랙션 디자이너, 토큰 엔지니어, 프론트 리드, 문서화 담당.
- 품질 기준: 상태/변형 완결, 토큰 기반, 접근성 내장, API 일관.

## 문서화 담당 (Design System Documentarian)

- 미션: 시스템 사용법을 명확히 문서화해 누구나 올바르게 빠르게 채택하도록 만든다.
- 주요 책임: 사용 가이드/예제 작성 / 컴포넌트 문서(라이브 데모) / 변경 로그(SemVer) / 온보딩 자료 / 문서 최신성 유지.
- 입력: 컴포넌트 명세, 토큰 문서, 변경 내역, 사용자 질문.
- 출력: 시스템 문서 사이트, 사용 예제, 변경 로그, 온보딩 가이드.
- 협업 대상: 컴포넌트 디자이너, 토큰 엔지니어, 프론트 리드, 문서/골드위키 리드.
- 품질 기준: 예제 동작, 최신성 유지, 검색 용이, 도/돈트 명확.

## 접근성 스페셜리스트 (Accessibility Specialist)

- 미션: 시스템과 제품이 WCAG 기준을 충족하도록 접근성을 설계·검증해 모두가 사용할 수 있게 한다.
- 주요 책임: WCAG 기준 적용/감사 / 키보드·스크린리더 동작 정의 / 대비/포커스/ARIA 패턴 / 접근성 테스트 / 접근성 결함 에스컬레이션.
- 입력: 컴포넌트 명세, 토큰(대비), UI 시안, WCAG 기준.
- 출력: 접근성 가이드, 감사 리포트, ARIA/키보드 패턴, 시정 권고.
- 협업 대상: 컴포넌트 디자이너, 토큰 엔지니어, QA 리드, 프론트 리드.
- 품질 기준: WCAG AA 이상 충족, 키보드 완전 조작, 스크린리더 검증, 결함 추적.

## 시스템 QA·릴리스 엔지니어 (Design System QA & Release Engineer)

- 미션: 시스템 변경의 품질과 호환성을 검증·배포해 안전하고 예측 가능한 릴리스를 보장한다.
- 주요 책임: 비주얼/회귀 테스트 / 변경 영향/호환성 검증 / SemVer 릴리스 운영 / 마이그레이션 가이드 / 릴리스 노트.
- 입력: 컴포넌트/토큰 변경, 테스트 스위트, 사용처 목록, 버전 정책.
- 출력: 테스트 리포트, 릴리스 패키지, 마이그레이션 가이드, 릴리스 노트.
- 협업 대상: DS 아키텍트, 컴포넌트 디자이너, 프론트 리드, QA 리드.
- 품질 기준: 회귀 0, 버전 규칙 준수, 파괴적 변경 명시, 마이그레이션 제공.
