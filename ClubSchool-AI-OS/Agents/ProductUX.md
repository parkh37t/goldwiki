# ProductUX — 프로덕트·UX팀 역할 카탈로그

> 제품 전략·서비스 기획·UX 리서치·정보구조·플로우를 담당하는 사람용 역할 카탈로그입니다.
> 실행 정본은 `../.claude/agents/product-strategy-lead.md`,
> `../.claude/agents/service-planning-lead.md`, `../.claude/agents/ux-research-lead.md`,

> [GoldWiki/UX](../GoldWiki/UX/UXStrategyFramework.md)입니다.

## 팀 개요

프로덕트·UX팀은 "무엇을 만들고 어떻게 경험하게 할 것인가"를 정의합니다. 비즈니스 요구와
사용자 니즈를 제품 전략·서비스 시나리오·정보구조·유저플로우로 구체화해 UI 디자인의
입력을 만듭니다. 모든 결정은 [UX 전략 프레임워크](../GoldWiki/UX/UXStrategyFramework.md)를
먼저 참조하고, 사용자 근거는 리서치팀과 공유합니다.

- 핵심 산출물: 제품 전략, PRD, 서비스 시나리오, 페르소나·여정맵, IA, 유저플로우, 와이어프레임
- 주요 협업 팀: 비즈니스 컨설팅팀, 리서치팀, UI 디자인팀, 빌드(프론트)팀
- 핵심 문서: [UX](../GoldWiki/UX/UXStrategyFramework.md), [Business](../GoldWiki/Business/README.md), [UI](../GoldWiki/UI/README.md)

## 프로덕트 오너 (Product Owner, PO)

- 미션: 제품 비전·우선순위·범위를 책임지고 비즈니스 가치와 사용자 가치를 제품 백로그로 전환한다.
- 주요 책임: 제품 비전·로드맵 / 백로그 우선순위 / PRD 작성 / 범위·트레이드오프 결정 / 스프린트 목표 정렬.
- 입력: 비즈니스 케이스, 가치 모델, 사용자 인사이트, 기술 제약.
- 출력: 제품 전략, 로드맵, PRD, 우선순위 백로그, 수용 기준.
- 협업 대상: BA·가치설계자, 서비스기획자, UI 디자인 리드, 프론트 리드.
- 품질 기준: 우선순위 근거 명확, 범위 추적 가능, 수용 기준 검증 가능.

## 서비스 기획자 (Service Planner)

- 미션: 사용자 여정 전반의 서비스 시나리오와 기능 구조를 설계해 제품을 실사용 흐름으로 구체화한다.
- 주요 책임: 서비스 시나리오·여정 설계 / 기능 정의·정책서 작성 / 화면 정의(스토리보드) / 예외/엣지 케이스 정의 / 운영 정책 설계.
- 입력: PRD, 페르소나·여정맵, 요구사항, 도메인 규칙.
- 출력: 서비스 시나리오, 기능 정책서, 화면 정의서, 정책/예외 규칙.
- 협업 대상: 프로덕트 오너, UX 리서처, IA, UI 디자이너.
- 품질 기준: 시나리오 일관, 예외 케이스 커버, 정책 모호함 0, 화면-기능 추적.

## UX 리서처 (UX Researcher)

- 미션: 사용자 행동·니즈를 검증해 디자인 결정이 추측이 아닌 근거 위에 서도록 한다.
- 주요 책임: UX 리서치 설계·수행 / 페르소나·여정맵 작성 / 사용성 테스트 / 인사이트→디자인 요건 변환 / 가설 검증.
- 입력: 사용자 리서치(리서치팀), 프로토타입, 분석 데이터, 리서치 질문.
- 출력: 페르소나, 사용자 여정맵, 사용성 테스트 리포트, 디자인 요건.
- 협업 대상: 사용자 리서처(리서치팀), 서비스기획자, IA, UI 디자인 리드.
- 품질 기준: 방법/표본 명시, 인사이트는 증거 인용, 요건은 실행 가능.

## 정보구조 설계자 (Information Architect, IA)

- 미션: 콘텐츠·기능을 사용자의 멘탈 모델에 맞게 구조화해 길 잃지 않는 탐색 경험을 설계한다.
- 주요 책임: 사이트맵·내비게이션 설계 / 콘텐츠 분류·라벨링 / 카드소팅·트리테스트 / 검색/필터 구조 / 구조 일관성 관리.
- 입력: 기능 목록, 사용자 멘탈 모델, 여정맵, 콘텐츠 인벤토리.
- 출력: 사이트맵, 내비게이션 모델, 라벨 체계, 콘텐츠 분류표.
- 협업 대상: UX 리서처, 서비스기획자, 유저플로우 디자이너, UI 디자이너.
- 품질 기준: 구조 일관·확장 가능, 라벨 명료, 사용자 검증(카드소팅) 반영.

## 유저플로우 디자이너 (User Flow / Interaction Flow Designer)

- 미션: 사용자가 목표를 달성하는 화면 간 흐름을 최소 마찰로 설계해 전환과 만족을 높인다.
- 주요 책임: 태스크 플로우·화면 흐름 설계 / 분기/예외 흐름 정의 / 와이어프레임 작성 / 전환 지점 최적화 / 플로우 검증.
- 입력: IA, 서비스 시나리오, 페르소나 목표, 기능 정책.
- 출력: 유저플로우 다이어그램, 와이어프레임, 분기 시나리오, 인터랙션 노트.
- 협업 대상: IA, 서비스기획자, UI 인터랙션 디자이너, 프론트 리드.
- 품질 기준: 플로우 완결성(모든 분기 처리), 마찰 지점 명시, 와이어 일관.

## 프로덕트 애널리스트 (Product Analyst)

- 미션: 제품 사용 데이터와 지표를 분석해 우선순위와 개선의 근거를 제공한다.
- 주요 책임: 제품 KPI/퍼널 정의 / 사용 데이터 분석 / 가설 실험(A/B) 설계 / 인사이트→백로그 반영 / 성과 측정.
- 입력: 사용 로그, KPI 트리, 가설, 비즈니스 목표.
- 출력: 제품 지표 대시보드, 퍼널 분석, 실험 리포트, 개선 권고.
- 협업 대상: 프로덕트 오너, 데이터 분석 리드, UX 리서처, 서비스기획자.
- 품질 기준: 지표 정의 명확, 분석 재현 가능, 권고는 데이터 근거.
