# BusinessConsulting — 비즈니스 컨설팅팀 역할 카탈로그

> 고객 비즈니스 문제를 정의하고 요구사항·프로세스·가치 구조로 전환하는 사람용 역할 카탈로그입니다.
> 실행 정본은 `../.claude/agents/business-analysis-lead.md`, 거버넌스·표준 정본은
> [GoldWiki/Business](../GoldWiki/Business/README.md)입니다.

## 팀 개요

비즈니스 컨설팅팀은 "무엇을, 왜, 어떤 가치로 만들 것인가"를 정의합니다. 고객의 현황(as-is)을
분석하고 목표(to-be)를 설계하며, 요구사항을 추적 가능하게 구조화해 제품·디자인·개발팀이
오해 없이 실행하도록 합니다. 모든 분석은 [Business](../GoldWiki/Business/README.md) 표준을 먼저
참조하고 결정은 [DecisionLog](../GoldWiki/DecisionLog/README.md)에 반영합니다.

- 핵심 산출물: 비즈니스 케이스, 요구사항 명세(BRD/SRS), as-is/to-be 프로세스, 가치 모델
- 주요 협업 팀: 영업·제안팀, 리서치팀, 프로덕트·UX팀, 경영/리더십팀
- 핵심 문서: [Business](../GoldWiki/Business/README.md), [RFP](../GoldWiki/RFP/RFPAnalysisFramework.md), [UX](../GoldWiki/UX/UXStrategyFramework.md)

---

## 비즈니스 애널리스트 (Business Analyst, BA)

- 미션: 고객의 비즈니스 문제와 목표를 데이터·인터뷰로 규명해 투자 대비 가치가 명확한 비즈니스 케이스를 만든다.
- 주요 책임: 현황·문제 정의 / KPI·성공지표 설정 / 비용·효익(ROI) 분석 / 이해관계자 매핑 / 우선순위 권고.
- 입력: RFP·고객 인터뷰, 리서치 인사이트, 현업 데이터, 경영 목표.
- 출력: 비즈니스 케이스, KPI 트리, ROI 모델, 이해관계자 맵.
- 협업 대상: 가치설계자, 도메인전문가, 제안총괄, 프로덕트 오너.
- 품질 기준: 가정·데이터 출처 명시, KPI 측정 가능, 권고는 우선순위·근거 동반.

## 프로세스 컨설턴트 (Process Consultant)

- 미션: 업무 프로세스의 비효율·병목을 진단하고 개선된 to-be 프로세스를 설계해 운영 가치를 높인다.
- 주요 책임: as-is 프로세스 매핑 / 병목·낭비 진단 / to-be 프로세스/자동화 설계 / 변화관리 영향 분석 / 효과 측정 설계.
- 입력: 업무 흐름·SOP, 현업 인터뷰, 운영 데이터, 자동화 기회.
- 출력: as-is/to-be 프로세스맵, 개선안, 변화관리 계획, 효과 지표.
- 협업 대상: 도메인전문가, BA, 서비스기획자, AI 자동화 리드.
- 품질 기준: 프로세스 표기 일관(BPMN 등), 병목 근거 제시, 개선 효과 정량화.

## 요구공학가 (Requirements Engineer)

- 미션: 이해관계자 니즈를 모호함 없이 추적 가능한 요구사항으로 구조화해 범위·검증의 기준을 만든다.
- 주요 책임: 요구사항 수집·도출 / 기능/비기능 분류 / 요구사항 추적표(RTM) 관리 / 충돌·중복 해소 / 수용 기준 정의.
- 입력: 비즈니스 케이스, 사용자/도메인 입력, RFP 요구사항, 제약사항.
- 출력: BRD/SRS, 요구사항 추적표, 수용 기준, 범위 정의.
- 협업 대상: BA, 프로덕트 오너, QA 리드, 프로세스 컨설턴트.
- 품질 기준: 요구사항은 단일·검증가능·추적가능, 중복/충돌 0, 수용 기준 명확.

## 도메인 전문가 (Domain / Subject Matter Expert, SME)

- 미션: 해당 산업·업무 도메인의 깊은 지식으로 솔루션의 현실 적합성과 규제 정합성을 보증한다.
- 주요 책임: 도메인 용어·규칙 정의 / 업종 규제/관행 검토 / 시나리오 타당성 검증 / 리스크/예외 케이스 식별 / 현업 검증 주선.
- 입력: 업종 자료([Industry](../GoldWiki/Industry/README.md)), 규제 요건, 현업 피드백, 요구사항 초안.
- 출력: 도메인 규칙집, 규제 체크리스트, 시나리오 검증 결과, 용어집 기여.
- 협업 대상: BA, 요구공학가, 리서치 업종전문가, 컴플라이언스 담당.
- 품질 기준: 도메인 정확성, 규제 최신성, 예외/리스크 명시.

## 가치설계자 (Value Architect / Value Designer)

- 미션: 솔루션이 만들어낼 비즈니스·사용자 가치를 구조화해 우선순위와 차별화의 근거를 제공한다.
- 주요 책임: 가치 제안(value proposition) 설계 / 가치-기능 매핑 / 가치 흐름(value stream) 정의 / 우선순위(가치×노력) / 가치 실현 지표 설계.
- 입력: 비즈니스 케이스, 사용자 니즈, 경쟁 포지셔닝, 기능 후보.
- 출력: 가치 제안 캔버스, 가치-기능 매트릭스, 우선순위 보드, 가치 KPI.
- 협업 대상: BA, 프로덕트 오너, BX 브랜드전략가, 제안총괄.
- 품질 기준: 가치-기능 추적 가능, 우선순위 근거 명확, KPI와 연결.

## 솔루션 컨설턴트 (Solution Consultant)

- 미션: 비즈니스 요구를 실현 가능한 솔루션 개념으로 번역해 기술팀과 비즈니스팀의 다리를 놓는다.
- 주요 책임: 솔루션 옵션 정의·비교 / feasibility 사전 검토 / 개념 아키텍처 스케치 / 트레이드오프 분석 / 솔루션 로드맵 권고.
- 입력: 요구사항, 가치 모델, 기술 제약([Backend](../GoldWiki/Backend/README.md)·[AI](../GoldWiki/AI/README.md)), 예산.
- 출력: 솔루션 옵션 비교표, 개념 아키텍처, 트레이드오프 분석, 권고안.
- 협업 대상: 요구공학가, CTO/기술 리드, 가치설계자, 프로덕트 오너.
- 품질 기준: 옵션은 비용/리스크/효익 비교, feasibility 검증, 권고 근거 명확.
