# QA 체크리스트 템플릿 (QA Checklist Template)

> **용도**: 산출물을 클라이언트 제출/납품 전 통과시키는 **10단계 품질 검증 체계**와 테스트케이스 추적표를 제공한다. 게이트형 검증으로, 미충족 항목이 하나라도 있으면 반려한다.
> **사용 에이전트**: qa-lead(주), 모든 산출 에이전트, pmo-director.
> **선행 산출물**: 전 단계 산출물(요구ID·화면ID·FLOW-ID 보유)
> **관련 GoldWiki**: [품질 리뷰 체크리스트](../GoldWiki/QA/QualityReviewChecklist.md) · [29 마스터 품질 체크리스트](../GoldWiki/29_QUALITY_CHECKLIST.md) · [30 테스트 전략](../GoldWiki/30_TEST_STRATEGY.md)

### 사용 안내
- 아래 **10단계**를 순서대로 통과시킨다. 각 단계는 게이트다.
- 모든 테스트케이스(TC-ID)는 **요구ID·화면ID·FLOW-ID**에 매핑해 추적성을 닫는다.
- 반복 결함은 [39 공통 오류](../GoldWiki/39_COMMON_ERRORS.md)에 누적한다.

## 2. 10단계 품질 검증 체계

> 각 단계 전부 통과해야 다음 게이트로 인계된다.

| 단계 | 검증 영역 | 핵심 질문 | 통과 | 비고 |
|------|-----------|-----------|------|------|
| 1 | 요구사항 충족 | 모든 요구ID가 구현·매핑됐는가 | ☐ | |
| 2 | 추적성 | 요구ID↔화면ID↔FLOW↔TC가 끊김 없이 연결되는가 | ☐ | |
| 3 | 기능 정확성 | 기능이 명세대로 동작하는가 | ☐ | |
| 4 | UX/사용성 | 핵심 플로우가 막힘 없이 완결되는가 | ☐ | |
| 5 | UI/디자인 정합 | 토큰·컴포넌트·반응형이 일치하는가 | ☐ | |
| 6 | 접근성 | WCAG 2.1 AA를 충족하는가 | ☐ | |
| 7 | 성능 | LCP/응답시간 등 목표를 만족하는가 | ☐ | |
| 8 | 보안 | OWASP·인증/인가·입력검증을 통과하는가 | ☐ | |
| 9 | 코드/문서 품질 | 표준·명명·문서화가 일관되는가 | ☐ | |
| 10 | 납품 준비 | 한국어 표현·산출물 완전성·게이트 승인 | ☐ | |

## 3. 단계별 상세 체크리스트

### 1단계 · 요구사항 충족
- [ ] 모든 Must 요구ID가 충족됐다.
- [ ] 미충족 요구는 사유·대안이 기록됐다.

### 2단계 · 추적성
- [ ] 요구ID마다 대응 화면ID가 있다.
- [ ] 화면ID마다 대응 FLOW가 있다.
- [ ] 요구/화면/플로우마다 검증 TC-ID가 있다.

### 3~4단계 · 기능 & UX
- [ ] 정상 경로가 완결된다.
- [ ] 빈/오류/로딩 상태가 처리된다.

### 5~6단계 · UI & 접근성
- [ ] 색상이 토큰에서 파생됐다.
- [ ] 키보드 전용 사용 가능, 대비 4.5:1.

### 7~8단계 · 성능 & 보안
- [ ] 성능 목표를 측정·충족했다.
- [ ] 인증/인가·입력검증·민감정보 처리 확인.

### 9~10단계 · 품질 & 납품
- [ ] 코드/문서 표준 준수.
- [ ] 자연스러운 한국어, 플레이스홀더 없음.
- [ ] 게이트 승인 완료.
