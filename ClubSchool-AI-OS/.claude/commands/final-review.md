# /final-review

## 목적
프로젝트 **최종 납품 직전**, 전체 산출물을 납품 체크리스트로 종합 검수한다. 완성도·품질·문서·계약 충족 여부를 점검하고, 납품 승인(Go) 또는 보완(No-Go)을 판정한다.

## 실행 조건
- 모든 단계 산출물이 통합된 납품 패키지가 존재한다.
- 계약/RFP의 납품 요건과 수용 기준이 확보되어 있다.

## 호출할 Subagent
- **qa-lead** (주담당) — 최종 검수·체크리스트 실행
- pmo-director — 계약/일정 충족 확인
- security-risk-lead — 보안/규정 최종 점검
- documentation-lead — 산출물/문서 완비 확인
- executive-director — 최종 납품 승인

## 입력값
- `$ARGUMENTS`: 납품 패키지 경로 또는 프로젝트명
- 부가: 계약 납품 요건, 수용 기준

## 처리 절차
1. **납품 범위 대조** — 계약/RFP 납품 요건과 실제 산출물을 1:1 대조한다.
2. **품질 종합 검수** — 기능·접근성·성능·보안의 최종 상태를 확인한다.
3. **문서 완비 확인** — 산출물 문서·매뉴얼·인수인계 자료를 점검한다.
4. **잔여 결함 점검** — Blocker/Critical 결함 0건을 확인한다.
5. **수용 기준 검증** — 각 수용 기준의 충족 증빙을 확인한다.
6. **납품 판정** — Go/No-Go를 판정하고 미충족 항목과 보완 계획을 명시한다.

## 출력값
```markdown
# 최종 납품 검수 — <프로젝트명>
## 1. 납품 범위 대조표
| 납품 요건 | 산출물 | 충족 | 증빙 |
## 2. 품질 종합 결과 (기능/접근성/성능/보안)
## 3. 문서/인수인계 완비 점검
## 4. 잔여 결함 현황
## 5. 수용 기준 충족 검증
## 6. 납품 판정 (Go/No-Go) + 보완 계획
```

## 품질 기준
- [ ] 모든 계약 납품 요건이 산출물과 대조 확인되었다.
- [ ] Blocker/Critical 결함이 0건이다.
- [ ] 모든 수용 기준에 충족 증빙이 있다.
- [ ] 인수인계 문서가 완비되었다.
- [ ] 납품 결과가 ProjectMemory·DecisionLog에 기록되었다.
- No-Go 시 결함을 해당 단계 커맨드(`/qa-review` 등)로 회귀시킨다.
