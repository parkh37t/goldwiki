# /design-start

## 목적
화면 요건과 브랜드 전략을 입력으로 **UI 디자인을 착수**한다. 아트디렉션(무드·컬러·타이포·그리드)을 정하고 핵심 화면 시안과 디자인 원칙을 수립해 디자인시스템 구축의 토대를 만든다.

## 실행 조건
- `/screenlist-create` 산출물(화면 목록·요건)이 존재한다.
- 브랜드 가이드 또는 브랜드 가설이 확보되어 있다.

## 호출할 Subagent
- **ui-design-lead** (주담당) — 아트디렉션·화면 시안
- bx-design-lead — 브랜드 경험 정렬
- ux-research-lead — 사용성 검증
- design-system-lead — 토큰화 가능성 검토

## 참조할 Gold Wiki 문서
- `../../GoldWiki/UI/UIDesignGuide.md` — UI 디자인 원칙(정본)
- `../../GoldWiki/Brand/BrandExperienceGuide.md` — 브랜드 경험 가이드
- `../../GoldWiki/08_UI_GUIDELINES.md` — UI 가이드라인
- `../../GoldWiki/10_FIGMA_GUIDE.md` — 디자인 산출 표준

## 입력값
- `$ARGUMENTS`: 화면 목록 산출물 경로 또는 프로젝트명
- 부가: 브랜드 가이드, 레퍼런스 무드보드(선택)

## 처리 절차
1. **아트디렉션 정의** — 무드·키워드·레퍼런스로 시각 방향을 정한다.
2. **컬러/타이포 시스템** — 1차 컬러 팔레트와 타입 스케일을 정한다.
3. **레이아웃/그리드** — 브레이크포인트별 그리드와 간격 체계를 정한다.
4. **핵심 화면 시안** — 대표 화면 3~5개의 고충실도 시안을 제작한다.
5. **디자인 원칙 수립** — 일관성·접근성·반응형 원칙을 정한다.
6. **토큰화 후보 식별** — 색/타입/간격 등 토큰화 대상을 추출한다.

## 출력값
```markdown
# UI 디자인 착수 — <프로젝트명>
## 1. 아트디렉션 (무드·키워드)
## 2. 컬러 / 타이포 시스템
## 3. 레이아웃 / 그리드
## 4. 핵심 화면 시안 (3~5)
## 5. 디자인 원칙
## 6. 토큰화 후보
```

## 품질 기준
- [ ] 아트디렉션이 브랜드 전략과 정합한다.
- [ ] 컬러 대비가 WCAG AA를 충족한다.
- [ ] 핵심 화면이 화면 요건/상태를 반영한다.
- [ ] 토큰화 후보가 디자인시스템 인계 형태로 정리되었다.
- [ ] DecisionLog가 갱신되었다.
- 통과 시 `/design-system-create`로 인계한다.
