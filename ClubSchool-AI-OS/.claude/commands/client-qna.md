# /client-qna

## 목적
제안 발표·심사·미팅을 앞두고 **고객이 던질 예상 질문과 답변**을 미리 준비한다. AIClientSimulation으로 다양한 이해관계자 페르소나의 날카로운 질의를 생성하고, 근거 기반의 설득 답변 스크립트를 만든다.

## 실행 조건
- 제안서 또는 발표 자료/산출물이 존재한다.
- 발주처 이해관계자 구성(의사결정자·실무자·예산권자)이 파악되어 있다.

## 호출할 Subagent
- **client-simulation-lead** (주담당) — AIClientSimulation 운영·질의 생성
- proposal-lead — 답변 정합성·메시지 정렬
- rfp-strategy-lead — 평가 관점 질의 보강
- security-risk-lead — 보안/규정 질의 대비

## 입력값
- `$ARGUMENTS`: 제안서/산출물 경로 또는 사업명
- 부가: 이해관계자 구성, 알려진 우려사항

## 처리 절차
1. **페르소나 구성** — 의사결정자·실무자·예산권자·기술검토자 페르소나를 구성한다.
2. **질의 생성** — 페르소나별로 우호/중립/적대 질문을 생성한다(가격·일정·리스크·차별화).
3. **난도/리스크 분류** — 질문을 답변 난도와 위험도로 분류한다.
4. **답변 스크립트** — 각 질문에 근거 기반 핵심 답변 + 보강 근거를 작성한다.
5. **함정 질문 대비** — 약점을 파고드는 질문에 대한 방어 논리를 준비한다.
6. **리허설 가이드** — 발표자별 분담과 톤 가이드를 제공한다.

## 출력값
```markdown
# 고객 예상 질의응답 — <사업명>
## 1. 이해관계자 페르소나
## 2. 예상 질의응답표
| # | 페르소나 | 질문 | 난도 | 핵심 답변 | 보강 근거 |
## 3. 함정 질문 / 방어 논리
## 4. 리허설 가이드 (분담·톤)
```

## 품질 기준
- [ ] 페르소나별로 충분한 질의가 생성되었다(우호/중립/적대 균형).
- [ ] 모든 답변에 근거가 연결된다.
- [ ] 약점/함정 질문에 대한 방어 논리가 있다.
- [ ] 답변이 제안 메시지와 일관된다.
- [ ] ProjectMemory에 Q&A가 기록되었다.
- 약점 노출 시 `/proposal-review`로 보완한다.
