# /ia-create

## 목적
사용자 정의와 UX 전략을 입력으로 **정보구조(IA)**를 설계한다. 사이트/앱의 계층 구조, 내비게이션, 콘텐츠 모델, URL/라우팅 체계를 정의해 이후 유저플로우·화면설계의 골격을 만든다.

## 실행 조건
- `/ux-start` 산출물(UX 착수 패키지)이 존재한다.
- 핵심 과업과 콘텐츠 유형이 식별되어 있다.

## 호출할 Subagent
- **information-architecture-lead** (주담당) — IA·내비게이션·콘텐츠 모델 설계
- ux-research-lead — 사용자 멘탈모델 검증
- service-planning-lead — 기능 범위 정렬
- backend-lead — 데이터/콘텐츠 모델 정합성

- `../../GoldWiki/11_INFORMATION_ARCHITECTURE.md` — IA 원칙

## 입력값
- `$ARGUMENTS`: UX 착수 패키지 경로 또는 프로젝트명
- 부가: 기능 목록, 콘텐츠 인벤토리(선택)

## 처리 절차
1. **콘텐츠 인벤토리** — 콘텐츠/기능 항목을 수집하고 분류한다.
2. **카드소팅 분석** — 사용자 멘탈모델에 맞는 그룹핑을 정한다.
3. **계층 구조 설계** — 사이트맵(트리)을 깊이·폭 균형으로 구성한다.
4. **내비게이션 정의** — 글로벌/로컬/유틸리티 내비게이션과 라벨링을 정한다.
5. **콘텐츠 모델링** — 핵심 엔티티·속성·관계를 정의한다.
6. **라우팅/URL 규칙** — URL 패턴과 상태(빈/오류/로딩)를 명시한다.

## 출력값
```markdown
# 정보구조(IA) — <프로젝트명>
## 1. 콘텐츠 인벤토리
## 2. 사이트맵 (트리/표)
## 3. 내비게이션 체계 / 라벨링
## 4. 콘텐츠 모델 (엔티티·속성·관계)
## 5. URL/라우팅 규칙
## 6. 빈/오류/예외 상태 정책
```

## 품질 기준
- [ ] 모든 핵심 과업이 3클릭 이내 도달 가능하다(또는 근거 있는 예외).
- [ ] 라벨링이 사용자 언어 기준으로 일관된다.
- [ ] 콘텐츠 모델이 백엔드 데이터 모델과 정합한다.
- [ ] 예외 상태가 누락 없이 정의되었다.
- [ ] DecisionLog가 갱신되었다.
- 통과 시 `/userflow-create`로 인계한다.
