# /design-system-create

## 목적
UI 디자인 시안을 입력으로 **디자인 시스템**을 구축한다. 디자인 토큰, 컴포넌트 라이브러리, 사용 가이드를 정의해 디자인-퍼블리싱-개발 간 단일 기준을 만들고 일관성과 재사용성을 보장한다.

## 실행 조건
- `/design-start` 산출물(아트디렉션·핵심 시안·토큰화 후보)이 존재한다.
- 컴포넌트로 추상화할 패턴이 식별되어 있다.

## 호출할 Subagent
- **design-system-lead** (주담당) — 토큰·컴포넌트·문서화
- ui-design-lead — 시각 일관성
- frontend-lead — 코드 컴포넌트 정합성
- publishing-lead — 마크업/접근성 구현
- documentation-lead — 사용 가이드 정리

## 입력값
- `$ARGUMENTS`: 디자인 착수 산출물 경로 또는 프로젝트명
- 부가: 기술 스택(컴포넌트 프레임워크), 네이밍 규칙

## 처리 절차
1. **토큰 정의** — 색/타입/간격/그림자/모션 등을 토큰으로 체계화한다(시맨틱/원시 분리).
2. **컴포넌트 인벤토리** — 원자→분자→유기체 단위로 컴포넌트를 정의한다.
3. **상태/변형 명세** — 컴포넌트별 상태·변형·반응형 규칙을 정의한다.
4. **접근성 규격** — 키보드·ARIA·대비 기준을 컴포넌트마다 명시한다.
5. **사용 가이드** — Do/Don't, 조합 규칙, 코드 스니펫을 작성한다.
6. **버전/거버넌스** — SemVer·변경 절차·기여 규칙을 정한다.

## 출력값
```markdown
# 디자인 시스템 — <프로젝트명>
## 1. 디자인 토큰 (원시/시맨틱)
## 2. 컴포넌트 인벤토리 (원자/분자/유기체)
## 3. 컴포넌트별 상태·변형·반응형
## 4. 접근성 규격
## 5. 사용 가이드 (Do/Don't·스니펫)
## 6. 버전/거버넌스 (SemVer)
```

## 품질 기준
- [ ] 토큰이 원시/시맨틱으로 분리되고 하드코딩 값이 없다.
- [ ] 모든 컴포넌트에 상태·접근성 규격이 정의되었다.
- [ ] 디자인-코드 컴포넌트가 1:1로 매핑된다.
- [ ] 버전/거버넌스 규칙이 명시되었다.
- [ ] ComponentLibrary·BestPractices가 갱신되었다.
- 통과 시 `/publish-start`로 인계한다.
