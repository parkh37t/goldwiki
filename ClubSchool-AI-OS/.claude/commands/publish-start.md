# /publish-start

## 목적
디자인 시스템과 화면 시안을 입력으로 **시맨틱 HTML/CSS 프로토타입**을 퍼블리싱한다. 접근성(WCAG)·반응형·성능 기준을 충족하는 마크업을 만들어 프론트엔드 개발의 직접 기반을 제공한다.

## 실행 조건
- `/design-system-create` 산출물(토큰·컴포넌트)이 존재한다.
- 화면 요건과 상태 정의가 확정되어 있다.

## 호출할 Subagent
- **publishing-lead** (주담당) — 시맨틱 마크업·반응형·접근성 구현
- design-system-lead — 토큰/컴포넌트 적용 검증
- frontend-lead — 컴포넌트 구조 정합성
- qa-lead — 접근성/크로스브라우저 점검

## 입력값
- `$ARGUMENTS`: 디자인 시스템 산출물 경로 또는 프로젝트명
- 부가: 지원 브라우저, 브레이크포인트, 성능 목표

## 처리 절차
1. **마크업 구조 설계** — 시맨틱 태그·랜드마크·문서 구조를 정한다.
2. **토큰→CSS 매핑** — 디자인 토큰을 CSS 변수로 매핑한다.
3. **컴포넌트 퍼블리싱** — 컴포넌트를 재사용 가능한 마크업으로 구현한다.
4. **반응형 구현** — 브레이크포인트별 레이아웃을 구현한다.
5. **접근성 적용** — 키보드 포커스·ARIA·대비·대체텍스트를 적용한다.
6. **성능/검수** — 이미지 최적화·렌더 성능·크로스브라우저를 점검한다.

## 출력값
```markdown
# 퍼블리싱 프로토타입 — <프로젝트명>
## 1. 마크업 구조 / 랜드마크
## 2. CSS 토큰 매핑
## 3. 컴포넌트 마크업 (스니펫)
## 4. 반응형 규칙
## 5. 접근성 적용 내역 (WCAG 체크)
## 6. 성능/브라우저 검수 결과
```

## 품질 기준
- [ ] 시맨틱 마크업·랜드마크가 올바르게 사용되었다.
- [ ] 키보드만으로 모든 기능이 조작 가능하다.
- [ ] 색 대비가 WCAG AA를 충족한다.
- [ ] 정의된 브레이크포인트에서 레이아웃이 깨지지 않는다.
- [ ] BestPractices·CommonErrors가 갱신되었다.
- 통과 시 `/frontend-plan`으로 인계한다.
