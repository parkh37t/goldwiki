# /wbs-create

## 목적
확정된 프로젝트 범위를 입력으로 **작업분해구조(WBS)**와 일정·RACI·마일스톤을 작성한다. 산출물 기준으로 작업을 분해하고 의존성과 책임을 명확히 해 실행 가능한 추진 계획을 만든다.

## 실행 조건
- 요구사항/제안 범위 또는 화면/기능 목록이 확정되어 있다.
- 가용 자원(역할·기간)이 식별되어 있다.

## 호출할 Subagent
- **pmo-director** (주담당) — WBS·일정·RACI·마일스톤
- proposal-lead — 제안 범위 정렬(제안 단계인 경우)
- frontend-lead / backend-lead / ui-design-lead — 작업 추정
- security-risk-lead — 일정 리스크 반영

## 참조할 Gold Wiki 문서

- `../../GoldWiki/27_AUTOMATION_WORKFLOW.md` — 단계·게이트 연계

- `../../GoldWiki/35_PROJECT_MEMORY.md` — 프로젝트 메모리

## 입력값
- `$ARGUMENTS`: 범위 산출물 경로 또는 프로젝트명
- 부가: 가용 자원, 시작일/마감일, 외부 의존성

## 처리 절차
1. **범위 분해** — 산출물 중심으로 WBS(레벨 1~3)를 구성한다.
2. **작업 추정** — 작업별 공수·기간·필요 역할을 추정한다.
3. **의존성 도출** — 선후행 관계와 크리티컬 패스를 식별한다.
4. **일정 배치** — 마일스톤과 게이트에 작업을 배치한다(간트/표).
5. **RACI 지정** — 작업/산출물별 R/A/C/I를 지정한다.
6. **버퍼/리스크 반영** — 일정 버퍼와 리스크 대응 시점을 반영한다.

## 출력값
```markdown
# WBS / 추진 계획 — <프로젝트명>
## 1. WBS (레벨 1~3)
## 2. 작업 추정표
| WBS ID | 작업 | 공수 | 기간 | 역할 | 선행 |
## 3. 마일스톤 / 게이트
## 4. RACI 매트릭스
## 5. 크리티컬 패스 / 일정 버퍼
```

## 품질 기준
- [ ] 모든 산출물이 WBS 항목으로 분해되었다(누락 0).
- [ ] 작업마다 공수·역할·선행이 지정되었다.
- [ ] 크리티컬 패스가 식별되었다.
- [ ] RACI에 책임자(A)가 단일 지정되었다.
- [ ] ProjectMemory·DecisionLog가 갱신되었다.
- 일정 리스크가 크면 `/risk-review`로 연계한다.
