# /proposal-review

## 목적
완성된 제안서를 **발주처 평가위원의 시선**으로 채점하고, 배점 대비 약점과 개선 우선순위를 도출한다. 제출 전에 승률을 끌어올리는 최종 리허설 역할을 한다.

## 실행 조건
- `/proposal-build` 산출물(제안서 초안)이 존재한다.
- 평가 배점표(평가항목·가중치)가 확보되어 있다.

## 호출할 Subagent
- **client-simulation-lead** (주담당) — 평가위원 시뮬레이션 / AIEvaluationBoard 운영
- proposal-lead — 지적 사항 반영 방향 검토
- rfp-strategy-lead — 평가 정렬 재확인
- qa-lead — 형식·완성도 점검

## 참조할 Gold Wiki 문서
- `../../GoldWiki/Business/AIEvaluationBoard.md` — 평가위원 시뮬레이션 보드 운영 기준
- `../../GoldWiki/Proposal/ProposalStrategyPlaybook.md` — 제안 평가 루브릭
- `../../GoldWiki/29_QUALITY_CHECKLIST.md` — 제안 품질 체크리스트
- `../../GoldWiki/34_CLIENT_KNOWLEDGE.md` — 발주처 성향

## 입력값
- `$ARGUMENTS`: 제안서 초안 경로 또는 사업명
- 부가: 평가 배점표, 발주처 성향 메모

## 처리 절차
1. **평가 보드 구성** — AIEvaluationBoard로 다수 평가위원 페르소나(기술/사업/예산/사용자)를 구성한다.
2. **항목별 채점** — 배점표 기준으로 각 항목을 점수화하고 근거를 기록한다.
3. **강·약점 분석** — 고득점/저득점 구간과 그 원인을 분석한다.
4. **감점 요인 적출** — 누락 요구·모호 표현·근거 부족·형식 오류를 적출한다.
5. **개선 우선순위화** — 점수 기여도×수정 난이도로 개선 항목을 우선순위화한다.
6. **승률 추정** — 예상 총점과 경쟁 대비 승률 가설을 제시한다.

## 출력값
```markdown
# 제안서 평가 결과 — <사업명>
## 1. 평가위원 페르소나 구성
## 2. 항목별 채점표
| 평가항목 | 배점 | 획득(추정) | 근거 | 개선 여지 |
## 3. 강점 / 약점
## 4. 감점 요인 (우선순위)
## 5. 개선 권고 (영향×난이도)
## 6. 예상 총점 / 승률 가설
```

## 품질 기준
- [ ] 모든 평가항목이 채점되고 근거가 기록되었다.
- [ ] 감점 요인마다 구체적 개선안이 제시되었다.
- [ ] 개선 권고가 영향도 기준으로 우선순위화되었다.
- [ ] 평가 결과가 ProjectMemory에 기록되었다.
- 약점 보완 후 재집필이 필요하면 `/proposal-build`로 회귀한다.
