# /rfp-analyze

## 목적
`/rfp-start`로 만든 1차 분석 패키지를 **제안서 집필이 가능한 수준**으로 심화한다. 요구사항 추적성 매트릭스(RTM)를 완성하고, 평가 배점 대비 우리의 대응 강도를 정량화하며, 숨은 기대와 리스크를 검증한다.

## 실행 조건
- `/rfp-start` 산출물(RFP 분석 패키지)이 존재한다.
- 미해결 질의(Q&A) 후보가 정리되어 있거나 발주처 사전 질의 응답이 확보되어 있다.

## 호출할 Subagent
- **rfp-strategy-lead** (주담당) — 분석 심화·전략 정렬
- business-analysis-lead — RTM 완성·요구 정량화
- industry-research-lead — 동종 사례 벤치마크
- security-risk-lead — 규정/보안 요구 검증
- client-simulation-lead — 발주처 관점 평가 시뮬레이션

## 입력값
- `$ARGUMENTS`: RFP 분석 패키지 경로 또는 사업명
- 부가: 발주처 사전 질의 응답, 경쟁 정보(선택)

## 처리 절차
1. **요구사항 재검증** — `/rfp-start`의 요구 표를 점검하고 누락·중복·모호 항목을 보정한다.
2. **추적성 매트릭스(RTM) 완성** — 요구 ID ↔ 평가항목 ↔ 제안 섹션 ↔ 담당 ↔ 증빙을 한 표로 연결한다.
3. **평가 대응 강도 정량화** — 항목별 우리 대응 수준을 상/중/하로 점수화하고 약점 구간을 식별한다.
4. **숨은 기대 검증** — `/rfp-start`의 가설을 근거로 채택/기각하고 윈테마에 반영한다.
5. **리스크 정밀화** — 리스크 레지스터를 갱신하고 완화·전가·수용 전략을 확정한다.
6. **질의(Q&A) 정리** — 발주처에 보낼 공식 질의 항목과 자체 가정을 분리한다.
7. **차별화 포인트 확정** — 경쟁 대비 우위 메시지 3~5개를 근거와 함께 확정한다.

## 출력값
```markdown
# RFP 심층 분석 결과 — <사업명>
## 1. 요구사항 추적성 매트릭스(RTM)
| 요구 ID | 평가항목 | 제안 섹션 | 대응 강도 | 증빙 | 담당 |
## 2. 평가 대응 강도 분석 (약점 구간 표시)
## 3. 검증된 숨은 기대 / Win Theme
## 4. 리스크 레지스터 (완화 전략 확정)
## 5. 발주처 질의 목록 / 자체 가정
## 6. 차별화 포인트 (근거 인용)
```

## 품질 기준
- [ ] RTM에 고아 요구·고아 평가항목이 0건이다.
- [ ] 모든 약점 구간에 보완 전략이 지정되었다.
- [ ] 숨은 기대 가설이 채택/기각 근거와 함께 처리되었다.
- [ ] 차별화 포인트마다 검증 가능한 근거가 있다.
- [ ] DecisionLog·ProjectMemory가 갱신되었다.
- 통과 시 `/proposal-build`로 인계한다.
