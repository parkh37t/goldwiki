# /screenlist-create

## 목적
사용자 플로우를 입력으로 **화면 목록(스크린 리스트)**과 화면별 요건 정의서를 작성한다. 각 화면의 목적·구성요소·상태·데이터·인터랙션을 명세해 UI 디자인과 퍼블리싱·개발의 공통 기준을 만든다.

## 실행 조건
- `/userflow-create` 산출물(사용자 플로우)이 존재한다.
- IA의 사이트맵·라우팅이 확정되어 있다.

## 호출할 Subagent
- **ui-design-lead** (주담당) — 화면 목록·요건 정의
- ux-research-lead — 과업/플로우 정합성
- information-architecture-lead — 라우팅 매핑
- frontend-lead — 구현 타당성 검토

## 참조할 Gold Wiki 문서
- `../../GoldWiki/UI/UIDesignGuide.md` — UI 설계 가이드(정본)
- `../../GoldWiki/08_UI_GUIDELINES.md` — UI 가이드라인
- `../../GoldWiki/UX/InformationArchitectureGuide.md` — IA/라우팅 연결
- `../../GoldWiki/12_USER_FLOW.md` — 플로우 연계

## 입력값
- `$ARGUMENTS`: 유저플로우 산출물 경로 또는 프로젝트명
- 부가: 디바이스/브레이크포인트 정책

## 처리 절차
1. **화면 추출** — 플로우의 각 노드를 화면으로 변환하고 중복을 통합한다.
2. **화면 ID 부여** — 라우트와 연결된 화면 ID(S-001…)를 부여한다.
3. **화면 요건 정의** — 화면별 목적·주요 요소·데이터·권한을 명세한다.
4. **상태 정의** — 기본/로딩/빈/오류/권한없음 상태를 화면마다 정의한다.
5. **인터랙션 명세** — 핵심 인터랙션과 전이 대상 화면을 연결한다.
6. **우선순위/범위** — MVP/후속 화면을 구분한다.

## 출력값
```markdown
# 화면 목록 / 요건 정의 — <프로젝트명>
## 1. 화면 목록표
| 화면 ID | 화면명 | 라우트 | 연결 과업 | 상태 종류 | 우선순위 |
## 2. 화면별 요건 정의서
### S-001 <화면명>
- 목적 / 주요 요소 / 데이터 / 권한 / 상태 / 인터랙션
## 3. 공통 컴포넌트 후보
## 4. MVP 범위
```

## 품질 기준
- [ ] 모든 플로우 노드가 화면으로 매핑되었다(누락 0).
- [ ] 화면마다 전 상태(로딩/빈/오류 등)가 정의되었다.
- [ ] 각 화면이 라우트·과업과 추적 가능하게 연결된다.
- [ ] 공통 컴포넌트 후보가 식별되었다(디자인시스템 인계용).
- [ ] DecisionLog가 갱신되었다.
- 통과 시 `/design-start`로 인계한다.
