# /ux-start

## 목적
프로젝트의 **UX 작업을 착수**해 사용자 정의, 핵심 문제, 리서치 계획, UX 전략의 기초를 마련한다. 이후 IA·유저플로우·화면설계의 출발점이 되는 단일 기준 문서를 만든다.

## 실행 조건
- 사업 목표/요구 정의(또는 RFP 분석 패키지)가 확보되어 있다.
- 대상 사용자군에 대한 1차 가설이 존재한다.

## 호출할 Subagent
- **ux-research-lead** (주담당) — 사용자 리서치·UX 전략 수립
- service-planning-lead — 서비스 목표 정렬
- product-strategy-lead — 프로덕트 전략 연결
- industry-research-lead — 업종 사용자 맥락

## 참조할 Gold Wiki 문서
- `../../GoldWiki/UX/UXStrategyFramework.md` — UX 전략 프레임워크(정본)
- `../../GoldWiki/07_UX_PRINCIPLES.md` — UX 원칙
- `../../GoldWiki/13_USER_JOURNEY.md` — 사용자 여정
- `../../GoldWiki/34_CLIENT_KNOWLEDGE.md` — 고객/사용자 지식

## 입력값
- `$ARGUMENTS`: 프로젝트명 또는 요구 정의 경로
- 부가: 비즈니스 목표(KPI), 기존 사용자 데이터(선택)

## 처리 절차
1. **목표 정렬** — 비즈니스 목표를 UX 성공지표(과업 성공률·이탈·만족도)로 번역한다.
2. **사용자 정의** — 핵심 페르소나 2~4개와 사용 맥락을 정의한다.
3. **핵심 문제 도출** — 사용자 Pain Point와 기회를 우선순위화한다.
4. **리서치 계획** — 정성/정량 조사 방법·표본·일정을 설계한다.
5. **UX 전략 수립** — 경험 원칙·차별화 방향·디자인 가설을 정한다.
6. **산출 로드맵** — IA → 유저플로우 → 화면설계 → 디자인 단계 계획을 연결한다.

## 출력값
```markdown
# UX 착수 패키지 — <프로젝트명>
## 1. UX 성공지표 (비즈니스 목표 연계)
## 2. 페르소나 / 사용 맥락
## 3. 핵심 문제 / 기회 (우선순위)
## 4. 리서치 계획
## 5. UX 전략 / 경험 원칙
## 6. UX 산출 로드맵
```

## 품질 기준
- [ ] UX 지표가 비즈니스 목표와 명확히 연결된다.
- [ ] 페르소나가 실제 의사결정에 쓸 수 있을 만큼 구체적이다.
- [ ] 핵심 문제가 근거와 함께 우선순위화되었다.
- [ ] 후속 단계(IA/플로우/화면) 인계 경로가 명시되었다.
- [ ] ProjectMemory가 갱신되었다.
- 통과 시 `/ia-create`로 인계한다.
