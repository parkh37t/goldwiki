# /executive-report

## 목적
복잡한 산출물(RFP 분석·제안·설계·리스크 등)을 **경영진이 30초에 결정할 수 있는 임원 보고서**로 압축한다. 핵심 결론·권고·리스크·필요 자원을 의사결정 중심으로 요약한다.

## 실행 조건
- 요약 대상 산출물이 존재한다.
- 의사결정 질문(승인/예산/일정 등)이 명확하다.

## 호출할 Subagent
- **executive-director** (주담당) — 임원 관점 요약·권고·승인
- pmo-director — 일정·자원 요약
- proposal-lead / rfp-strategy-lead — 사업 맥락(해당 시)
- documentation-lead — 형식·일관성

## 참조할 Gold Wiki 문서

- `../../GoldWiki/02_BUSINESS_GOALS.md` — 사업 목표

## 입력값
- `$ARGUMENTS`: 요약 대상 산출물 경로 또는 프로젝트명
- 부가: 의사결정 질문, 대상 독자(경영진/이사회)

## 처리 절차
1. **핵심 결론 추출** — 가장 중요한 결론/권고 1~3개를 맨 앞에 배치한다.
2. **현황 요약** — 진행 상태·핵심 수치(예산/일정/지표)를 압축한다.
3. **의사결정 항목** — 경영진이 결정해야 할 항목과 옵션·트레이드오프를 제시한다.
4. **리스크/이슈** — 상위 리스크와 대응 상태를 요약한다.
5. **자원/예산** — 필요 자원·예산·일정 영향을 명시한다.
6. **권고/다음 단계** — 명확한 권고와 기한이 있는 다음 행동을 제시한다.

## 출력값
```markdown
# 임원 보고서 — <프로젝트명>
## 핵심 결론 (TL;DR)
## 1. 현황 요약 (예산·일정·지표)
## 2. 의사결정 요청 (옵션·트레이드오프)
## 3. 상위 리스크 / 대응
## 4. 필요 자원 / 예산
## 5. 권고 / 다음 단계 (기한)
```

## 품질 기준
- [ ] 결론이 문서 최상단에 한눈에 들어온다.
- [ ] 의사결정 항목에 옵션·트레이드오프가 제시되었다.
- [ ] 모든 수치에 출처/근거가 연결된다.
- [ ] 분량이 1~2페이지를 넘지 않는다.
- [ ] DecisionLog가 갱신되었다.
- 승인/반려 결과를 ProjectMemory에 기록한다.
