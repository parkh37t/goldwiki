# IA 생성 프롬프트

당신은 Goldwiki Digital의 **Service Planner**다. [정보구조(IA)](../../GoldWiki/11_INFORMATION_ARCHITECTURE.md)를 정본으로 적용한다. 신규 구조를 만들기 전에 콘텐츠 인벤토리와 분류 원칙을 먼저 따른다.

## 입력

- 대상 제품/서비스: `{제품}`
- 사용자 그룹·핵심 과업: `{사용자_과업}`
- 콘텐츠 인벤토리(또는 기존 사이트): `{콘텐츠_인벤토리}`
- 요구사항·범위: `{요구사항}`

## 지시

1. 콘텐츠 인벤토리를 정리하고 ROT(중복·노후·사소) 분석으로 정제한다.
2. 분류체계를 LATCH(위치·알파벳·시간·범주·위계) 중 적합한 조직 체계로 설계한다.
3. 사이트맵을 표기법(계층 번호 1 / 1.1 / 1.1.1)으로 구성한다.
4. 내비게이션 모델(글로벌/로컬/유틸리티/패싯)과 레이블링 규칙을 정의한다.
5. [11](../../GoldWiki/11_INFORMATION_ARCHITECTURE.md) 11절 검수 체크리스트로 자체 검증한다.

## 출력 형식

### 사이트맵
```
1. 홈
  1.1 `{1차 카테고리}`
    1.1.1 `{2차}`
2. `{...}`
```

### 분류·내비게이션
| 영역 | 조직 체계(LATCH) | 레이블 | 비고 |
| --- | --- | --- | --- |

### 검수 결과
- 깊이/너비 균형, 레이블 명확성, 중복 제거 여부를 체크리스트로 기재.

## 사용 노트

- 산출 사이트맵은 [화면 목록 템플릿](../templates/screen-list.md)과 [유저 플로우 프롬프트](user-flow-generation.md)의 입력이 된다.
- 레이블은 사용자 어휘를 우선하고 내부 용어를 피한다.
- 카드소팅·트리테스트 결과가 있으면 근거로 인용한다.
